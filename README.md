# Sign-Language-interpreter
壹、系統簡介

一、規格目的

本系統旨在提供台灣手語 (TSL) 與中文（文字/語音)之間的即時雙向翻譯，促進聽障人士與一般人士在日常生活中的溝通。透過影像辨識、自然語言處理與 3D 動畫技術，本系統能夠：
1.將使用者的 TSL 手語動作即時辨識並轉換為中文文字或語音。
2.將中文文字或語音輸入即時轉換為對應的 TSL 手語動畫。

3.提供直觀的操作介面，縮短聽障與聽人之間的溝通障礙。
二、開發背景

現有溝通工具多依賴文字輸入或視訊聯繫，對於以手語為母語的聽障族群而言，轉換至文字邏輯仍存在門檻。目前市面上的手語辨識多針對美國手語 (ASL)，台灣手語 (TSL) 的即時翻譯工具相對匱乏。此外，手語的語法結構與口語中文存在顯
著差異，需要更深層的語意轉換機制。

三、開發動機

本專案的核心動機在於開發一個屬於台灣本土的雙向翻譯平台，讓手語使用者能以熟悉的肢體語言表達，同時讓不具備手語能力的聽人能透過語音或文字無障礙地理解對方的意思，進而提升醫療、櫃台服務及日常生活的互動品質。

四、開發需求

建置一套具備 TSL 與中文即時雙向翻譯 能力的系統。系統必須能精準擷取手部與臉部特徵，並處理手語語序與自然語言之間的轉換，確保翻譯內容的準確性。

五、系統特色與創新性

即時雙向翻譯：支援「手語錄入轉文字」與「文字輸入轉手語」雙向並行。
多模態特徵整合：利用 MediaPipe 結合影像辨識 (CNN) 擷取手部、臉部與姿態資料。
語意重組技術：透過 NLP 技術處理手語與中文語法間的轉換，避免生硬的字面直譯。
擬真 3D 視覺化：使用 3D 虛擬角色呈現手語，比傳統 2D 圖片更具表達力。

六、規格範圍  

1. 台灣手語辨識模組 (TSL → 文字/語音)

• 使用影像辨識技術即時識別手部與臉部動作。 
• 支援常用單詞、短語及日常會話句型的辨識。 
• 使用文本轉換技術轉換手語與中文語系之間的語序差異。 

2. 文字/語音轉手語模組 

• 將文字或語音內容轉換為對應 TSL 手語動畫。

貳、系統概述

一、系統目標

1.建立 TSL 手語動作的即時辨識與文字/語音輸出模組。

2.建立文字/語音內容轉換為 TSL 手語動畫的生成模組。

3.減少翻譯過程中的延遲，達成自然流暢的雙向對話體驗。

二、預計目標

本研究預計開發一套台灣手語雙向翻譯系統，透過 MediaPipe 擷取手勢與臉部特徵，由 AI 模型自動辨識並轉換成中文文字；反之，當接收到中文語音或文字時，系統將驅動 3D 虛擬角色生成對應手語動畫。
三、系統範圍 

1. 手語辨識模組

• 台灣手語→ 文字/語音 

• 支援日常會話、常用單詞與短語 

2. 文字/語音轉手語模組

文字或語音 → 台灣手語動畫 

動作與表情同步播放 

三、系統架構

本系統架構由以下核心模組組成：

1.特徵擷取模組 (MediaPipe)：偵測 21 點手部骨架、臉部表情及身體姿態。

2.手語辨識模組 (TSL Recognition)：將特徵點時序資料轉換為中文詞彙或句子（採用 Transformer 或 LSTM 模型）。

3.自然語言處理模組 (NLP)：處理中文語句與手語語序之間的映射轉換。

4.3D 手語動畫生成模組 (Blender/Three.js)：根據辨識結果驅動 3D 角色動作庫，顯示對應手語。

5.使用者介面層：提供簡單易用的視覺化視窗，顯示視訊畫面、翻譯結果與手語動畫。

2. 各模組功能說明

(1) 使用者介面（前端） 
提供語音、文字、影像輸入，並顯示翻譯後文字與3D Avatar手語動畫。使用Web技術（HTML、JavaScript、Three.js實作。 

(2) MediaPipe 特徵擷取模組 

利用 MediaPipe Hands、Face Mesh、Holistic 進行： 

• 21 點手部骨架偵測  

• 臉部表情與口型偵測 

• 全身姿態骨架偵測 

輸出統一格式之多模態關鍵點資料（JSON）。 

(3) 手語辨識模組（TSL） 

將 MediaPipe 的骨架資料輸入 AI 模型： 

• 使用者比 TSL → 模型輸出中文文字 

模型可採用： 

• Transformer 

• LSTM   

• 3D CNN + Temporal Attention 

(4) 自然語言處理模組（NLP） 

• POS 標註  

• 語法樹建立  

• 手語語序 ↔ 口語語序轉換  

產生最終的語言輸出文字。 

(5) 字幕／動畫同步模組  

此模組負責讓文字字幕與 3D 手語動畫保持一致時間點播放  

三大功能：  

• 動畫時間軸解析  

讀取每段手語 clip 的開始/結束時間。  

• 字幕對應引擎  

將文字斷句（或 gloss 分詞）與動畫片段配對。  

• 同步控制器  

3D Avatar 播到哪裡 → 字幕自動切換  

字幕換到下一句 → 動畫跳到對應 clip  

(6) 3D 手語動畫生成模組（Blender）  

使用 Blender 做：  

• 骨架綁定（Armature Rig）  

• 手語動作 Keyframe 自動生成  

• Clip 匯出為 glTF/FBX  

• 前端 Three.js 讀取播放  

此模組可由 AI 自動產 Keyframe，也可 manual 微調。

參、軟/硬體建構項目需求

一、功能需求

手語輸入與辨識：

即時偵測手部 21 個關鍵點及臉部特徵（如眉毛、嘴型）。

支援台灣手語日常詞彙與短語辨識。

虛擬角色手語生成：

根據中文文本，從動作庫調用對應的 TSL 手語片段進行組合播放。

動畫需包含臉部表情以符合手語表達習慣。

語音與文字轉換：

整合 ASR (自動語音辨識) 將人聲轉為文字，再轉為手語。

整合 TTS (文字轉語音) 將手語辨識出的文字朗讀出來。

二、介面需求 (UI)

介面設計採取直覺式配置：

左側:顯示即時攝影機畫面

中間:顯示 3D 手語動畫角色

右側：提供文字輸入框與語音錄入按鍵和顯示對話紀錄與翻譯結果（中文)。

三、軟/硬體環境

軟體：Python 3.8+、MediaPipe、PyTorch、Blender (動作庫製作)、Three.js (前端呈現)。

硬體：具備 GPU (建議 NVIDIA RTX 30 系列) 的筆電或電腦、高畫質攝影鏡頭、麥克風。

四、作業程序需求 

1. 使用者以手語或語音輸入內容。 

2. 系統進行影像辨識與自然語言分析。 

3. 輸出對應的文字與語音翻譯結果。 

4. 翻譯過程中維持即時性與流暢度。

五、其他需求 

1. 效能需求： 本系統為即時翻譯平台，為確保溝通流暢，整體翻譯運算
延遲（Latency）建議控制在 2~3 秒內；且 3D 動畫渲染幀率（FPS）
需維持在 30 FPS 以上，以確保手語動作清晰可辨。 

2. 隱私與安全性： 系統運作期間所擷取之使用者影像，僅用於即時特徵
點提取，不得在伺服器端永久儲存原始視訊檔案，以符合個資保護原
則。 

3. 可用性需求： 考量聽障人士需求，使用者介面（UI）需具備清晰的文
字顯示區，且在辨識失敗或環境光線不足時，提供視覺化的狀態提示。 

4. 擴充性需求： 手語動作資料庫（Motion Library）與語意分析模組應採
模組化設計，以利未來新增手語詞彙或支援其他語系。 

5. 語言準確性： 系統生成之翻譯內容與動畫，需符合 TSL之文法
結構（如語序倒裝、表情配合），避免因直譯造成語意誤解。

伍、預計貢獻

1.強化日常溝通自主性：提升聽障者在日常社交、門診醫療或臨櫃服務等公共情境中的即時溝通能力，減少對同步手語翻譯員的依賴。

2.完備雙向翻譯技術鏈：整合「手語辨識」與「動畫生成」技術，補足現有工具多為單向轉換（僅辨識或僅查詢）的不足，實現真正的對等溝通。

3.建立本土手語 AI 技術基礎：針對台灣手語（TSL）建立標準化的資料處理流程、關鍵點特徵抽取標準與模型訓練策略，為未來 TSL NLP（自然語言處理）與相關 AI 應用提供可靠的技術架構。

4.推動科技助殘與社會共融：促進資訊科技與社會福利的跨領域整合，透過技術手段消除溝通門檻，提升多樣化服務情境下的友善度與品質。
